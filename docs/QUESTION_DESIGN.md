# 题目设计指南

本文档说明 Resonance 题目库的设计原则，供贡献者参考。

## 核心原则

### 1. 没有"正确答案"

所有题目必须满足：**每个选项都代表真实存在、有人真诚持有的生活取向**。

❌ 错误示例：
> 当伴侣犯错时，你会：
> - A. 原谅TA，因为爱就是包容
> - B. 直接提出来，希望TA改正
> - C. 记在心里，等之后再说
> - D. 什么都不做

（"原谅"看起来是好答案，"记在心里"看起来是坏习惯）

✅ 正确示例：
> 当你们在家务分配上出现矛盾，你更倾向于：
> - A. 尽快把这个话题谈清楚
> - B. 等双方情绪都平稳后再谈
> - C. 暂时调整自己的期望，观察一段时间
> - D. 先做完眼前的，分配问题以后再说

### 2. 测量倾向，不测量能力

题目测量的是**价值观、优先级、处事风格**，而非知识水平、智力或道德高下。

❌ 避免：知道正确答案的人（如理财达人）比不知道的人得分更高
✅ 目标：两种截然不同的人生哲学，各自有其内在逻辑

### 3. 具体场景，不是抽象表态

❌ 模糊：你认为家庭应该排在事业之前吗？
✅ 具体：当一个重要的工作机会和家庭旅行冲突时，你会……

抽象问题更容易引发社会期望偏差（人们倾向于说"正确"的话，而不是真实的做法）。

### 4. 选项之间有真实张力

四个选项应该代表在某个维度上的不同位置，且每个选项内部逻辑自洽。

测试方法：能否为每个选项写出一个合理的持有者画像？如果某个选项你无法想象有人真心持有，考虑重写。

### 5. 分值反映维度光谱

每个维度有一个定义好的光谱（见各题库文件顶部注释）。分值从低到高对应光谱从一端到另一端：

- 低分（1–4）= 光谱一端
- 中间（5–7）= 中间立场
- 高分（8–10）= 光谱另一端

两端都是合理的，**分值差异代表分歧，不代表对错**。

## 题量结构

每个维度应有 **11 题**（情侣版）或 **9 题**（已婚版），按层级分布：

| Tier | 题数 | 选题策略 |
|------|------|----------|
| Quick | 前 3 题 | 最核心、最具代表性的场景 |
| Standard | 前 7 题（4–6 题为此层新增） | 扩展覆盖面 |
| Full | 全部 11/9 题 | 所有题目 |

**Quick tier 的题目最重要** — 它们必须能独立代表整个维度。

## Deal-breaker 题目

以下情况可以考虑标记 `weight: 2`：

- 分歧在现实中**几乎无法通过协商解决**（如：是否要孩子）
- 该题反映的是**价值观底线**，而非偏好或习惯
- 历史数据或研究表明该维度分歧是关系破裂的主要原因

使用 deal-breaker 权重需在 PR 中说明理由。

## 版本适配

### 情侣版题目

- 使用将来时、假设性场景："如果……你会……"
- 关注**预期和倾向**
- 避免需要有共同生活经验才能回答的问题

### 已婚版题目

- 使用现在时、过去时："你们通常……"、"上一次……你是"
- 关注**实际行为模式**
- 问题假设两人已经有一定共同生活经历

## 提交流程

1. 在 GitHub 上使用 [新问题模板](../.github/ISSUE_TEMPLATE/new-question.md) 提交 Issue
2. 维护者审核（关注点：无偏答案、分值合理性、场景真实性）
3. 通过后提交 PR，修改对应的 `src/data/questions/` 文件
4. 至少一名维护者 review 后合并

## 不适合提交的内容

- 测试人口统计特征的题目（年龄、教育背景、文化背景）
- 有明显政治立场的题目
- 需要特定文化背景才能理解的题目（目标受众为普通话使用者，但应尽量通用）
- 与现有题目高度重叠的题目
